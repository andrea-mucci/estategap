# Implementation Plan: AI Conversational Search UI

**Branch**: `021-ai-chat-search-ui` | **Date**: 2026-04-17 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/021-ai-chat-search-ui/spec.md`

---

## Summary

Build the primary entry point of the EstateGap frontend: a search-engine-style home page that transitions into a full AI chat window. The implementation spans a Zustand store for session management, a WebSocket hook for real-time streaming, nine chat UI components, inline listing search results with a map toggle, voice input with a Whisper fallback, and a conversation history sidebar. All work is purely frontend (TypeScript + Next.js 15) and builds on the `020-nextjs-frontend-foundation` layer.

---

## Technical Context

**Language/Version**: TypeScript 5.5 (strict mode) / Node.js 22  
**Primary Dependencies**: Next.js 15 (App Router), Tailwind CSS 4, shadcn/ui, Zustand 5, TanStack Query v5, next-intl, react-markdown + remark-gfm, @tailwindcss/typography, MapLibre GL JS  
**Storage**: sessionStorage (Zustand persist middleware) — no direct DB access  
**Testing**: Vitest + React Testing Library (unit); Playwright (E2E)  
**Target Platform**: Web browser — Chrome 120+, Edge 120+, Safari 17+ (desktop + iOS); Firefox 120+ (Whisper fallback path)  
**Project Type**: Frontend feature module within Next.js 15 monorepo  
**Performance Goals**: Streaming ≥ 30fps; listing results visible within 2s of criteria confirmation; voice transcription available within 500ms of recognition end  
**Constraints**: No new backend services; all state via WebSocket + REST; sessionStorage only (no IndexedDB)  
**Scale/Scope**: Single-user session scope; up to 200 messages per conversation; up to 50 listings per page

## Implementation Phases

### Phase 1 — Foundation (Types + Store + WebSocket Hook)

**Goal**: Shared types, Zustand store, WebSocket hook. Nothing visual yet.

**Files**:
- `frontend/src/types/chat.ts` — all interfaces from `data-model.md`
- `frontend/src/stores/chatStore.ts` — Zustand store with persist middleware
- `frontend/src/hooks/useChatWebSocket.ts` — connect, message dispatch, 50ms streaming buffer

**Key implementation details**:

1. **chatStore.ts**
   - Use `create` + `persist` with a custom `sessionStorage` storage adapter
   - `sessions` is a `Map<string, SessionState>`; convert to/from plain object for serialisation
   - `appendChunk` accumulates to the streaming message's `content` field directly (no copy)
   - `confirmSearch` sets status to `'confirmed'` and triggers TanStack Query in `SearchResults`

2. **useChatWebSocket.ts**
   - Accept `sessionId` and `jwt` as parameters
   - On mount: `new WebSocket(url + '?token=' + jwt)`
   - Chunk buffer: `let buffer = ''`; `setInterval(() => { if (buffer) { store.appendChunk(…); buffer = '' }; }, 50)`
   - `requestAnimationFrame` wrapper around each store dispatch
   - Reconnect: exponential backoff 1→2→4→8→max 30s (reuse existing WS backoff from `020`)
   - Dispatch all `IncomingWSMessage` types to the appropriate store actions

**Tests**: Unit tests for store actions (pure functions); hook tested with a mock WebSocket server via `vi.mock`.

### Phase 2 — Home Page & Chat Input

**Goal**: Home page with centred search input and mic button. Submitting a query navigates to `/chat`.

**Files**:
- `frontend/src/app/[locale]/page.tsx` — home page layout
- `frontend/src/components/chat/ChatInput.tsx` — auto-resize textarea
- `frontend/src/components/chat/TypingIndicator.tsx` — CSS keyframe dots
- i18n keys: `messages/{locale}.json` → `chat.placeholder`, `chat.send`, `chat.mic`

**Key implementation details**:

1. **page.tsx** — full-viewport centred flex column; `<ChatInput>` as the hero element; `<ConversationSidebar>` as a floating panel on desktop (hidden on mobile)
2. **ChatInput.tsx**
   - `<textarea>` with `rows={1}` and `onInput` → adjust `scrollHeight`
   - `onKeyDown`: Enter submits (unless Shift held); Shift+Enter inserts newline
   - `onSubmit`: calls `chatStore.createSession()` if no activeSession, then `chatStore.addMessage()`, sends `chat_message` over WS, navigates to `/chat`
   - Mic button renders `<VoiceInput>` in a modal/popover
   - Send button shows `<Loader2>` spinner when `isStreaming`

### Phase 3 — Voice Input

**Goal**: Full voice input pipeline with Web Speech API and Whisper fallback.

**Files**:
- `frontend/src/hooks/useVoiceInput.ts` — state machine + API selection
- `frontend/src/components/chat/VoiceInput.tsx` — visual feedback

**Key implementation details**:

1. **useVoiceInput.ts**
   - `const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition`
   - `isSupported`: `typeof SpeechRecognition !== 'undefined'`
   - State machine: `idle → listening → processing → idle` (or `error`)
   - Web Speech path: `recognition.continuous = false; recognition.interimResults = true; recognition.lang = locale`
   - Silence timer: 2000ms `setTimeout` reset on each `onresult`; calls `recognition.stop()` on expiry
   - Whisper path: `MediaRecorder` → `Blob` → `FormData` → `fetch('/api/whisper-proxy', { method: 'POST', body })`
   - Returns `{ state, transcript, interimTranscript, start, stop }`

2. **VoiceInput.tsx**
   - `listening` state: mic icon with `animate-pulse` Tailwind class
   - `processing` state: spinner
   - Waveform visualisation: use `AnalyserNode` from Web Audio API for amplitude bars
   - On `transcript` set → calls parent `onTranscript(transcript)` callback

3. **Whisper proxy** (`frontend/src/app/api/whisper-proxy/route.ts`): Next.js route handler that forwards the audio blob to OpenAI Whisper. Keeps the API key server-side.

### Phase 4 — Chat Window & Message Components

**Goal**: Full chat window with all message component types rendering correctly.

**Files**:
- `frontend/src/app/[locale]/chat/page.tsx`
- `frontend/src/components/chat/ChatWindow.tsx`
- `frontend/src/components/chat/MessageBubble.tsx`
- `frontend/src/components/chat/ChipSelector.tsx`
- `frontend/src/components/chat/ImageCarousel.tsx`
- `frontend/src/components/chat/CriteriaSummaryCard.tsx`

**Key implementation details**:

1. **ChatWindow.tsx**
   - `useEffect` on `messages.length` → scroll container ref to `scrollTop = scrollHeight`
   - Renders `<MessageBubble>` for each message
   - After last assistant message: if `isStreaming`, renders `<TypingIndicator>` (three dots) while waiting for first chunk

2. **MessageBubble.tsx**
   - User: `justify-end`, brand-colour background, no markdown
   - Assistant: `justify-start`, gray background, `<ReactMarkdown remarkPlugins={[remarkGfm]} className="prose prose-sm">` wrapper
   - After text content, renders `attachments.map(a => <AttachmentRenderer key={…} attachment={a} />)`

3. **ChipSelector.tsx**
   - `chips.map(chip => <Button variant="outline" size="sm" onClick={() => sendChip(chip.label)}>)`
   - `sendChip`: calls `useChatWebSocket().send({ type: 'chat_message', … })`

4. **ImageCarousel.tsx**
   - Container: `overflow-x-auto scroll-smooth snap-x snap-mandatory flex gap-3`
   - Each card: `snap-start min-w-[280px]` with image, price, location, Like/Not buttons
   - Like/Not: `useChatWebSocket().send({ type: 'image_feedback', … })`

5. **CriteriaSummaryCard.tsx**
   - shadcn `<Card>` with a 2-column grid of criteria fields
   - Each field: read mode shows label + value + pencil icon; click pencil → inline `<input>` or `<select>`
   - On blur/Enter: dispatch `chatStore.updateCriteria(sessionId, { [key]: newValue })`
   - "Search + Alert" `<Button>`: calls `chatStore.confirmSearch(sessionId)` → sends `criteria_confirm` over WS

### Phase 5 — Search Results

**Goal**: Inline listing cards, infinite scroll, map toggle, sort controls.

**Files**:
- `frontend/src/components/search/SearchResults.tsx`
- `frontend/src/components/search/ListingCard.tsx`
- `frontend/src/components/search/MapView.tsx`

**Key implementation details**:

1. **SearchResults.tsx**
   - Mounts when `session.status === 'confirmed'`
   - `useInfiniteQuery({ queryKey: ['listings', criteria], queryFn: fetchListings, getNextPageParam: r => r.nextCursor })`
   - Intersection Observer on a sentinel `<div>` at the bottom → calls `fetchNextPage()`
   - Sort controls: local `sortBy` state; passed as query param; invalidates query on change
   - Map toggle: local `showMap` boolean; renders `<MapView>` or card grid

2. **ListingCard.tsx**
   - Photo: `next/image` with `width=320 height=200` and `object-cover`
   - Deal score badge: `dealScore >= 70 → bg-green-500`, `40-69 → bg-amber-500`, `< 40 → bg-red-500`
   - Price formatted with `Intl.NumberFormat(locale, { style: 'currency', currency })`

3. **MapView.tsx**
   - MapLibre GL JS `new Map({ container: ref.current, style: '...' })`
   - `useEffect` on `listings` → remove + re-add GeoJSON source and layer
   - Click on pin → shows `<ListingCard>` popup

---

### Phase 6 — Conversation Sidebar

**Goal**: Sidebar listing recent conversations; click to load.

**Files**:
- `frontend/src/components/chat/ConversationSidebar.tsx`

**Key implementation details**:

1. `useQuery({ queryKey: ['chat-sessions'], queryFn: fetchSessions, staleTime: 30_000 })`
2. Each item: snippet (first 100 chars), relative date (`date-fns formatDistanceToNow`), status badge (shadcn `<Badge>`)
3. Click: `chatStore.loadSession(sessionId)` → navigates to `/chat` if not already there
4. On mobile: sidebar is a slide-over `<Sheet>` (shadcn); triggered by a hamburger button in `ChatWindow`

## Risk Register

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Web Speech API quirks on iOS Safari | Medium | High | Test on real device early; Whisper fallback covers gaps |
| MapLibre GL JS bundle size (≈ 600KB) | Low | Medium | Dynamic import with `next/dynamic`; only loads on map toggle |
| Zustand Map serialisation to sessionStorage | Low | Medium | Custom storage adapter tested in Phase 1 unit tests |
| 50ms buffer causing visible delay on fast LLMs | Low | Low | Buffer duration is configurable via a constant; tune if needed |
| `019-ws-chat-realtime` message protocol drift | Medium | High | Validate against `contracts/websocket-protocol.md`; add Zod parse at WS layer |
