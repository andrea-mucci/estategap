# Implementation Plan: EU Portals & Enrichment

**Branch**: `025-eu-portals-enrichment` | **Date**: 2026-04-17 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/025-eu-portals-enrichment/spec.md`

---

## Summary

Extend the EstateGap spider and enrichment pipeline to cover 4 new European markets (Italy, France, UK, Netherlands) via 7 new portal spiders and 4 country-specific enrichers. Each spider follows the existing `BaseSpider` pattern (Python, NATS JetStream, auto-registry). Enrichers follow the `BaseEnricher` pattern (PostgreSQL/PostGIS for spatial matching, WFS API for Dutch BAG data). Three new PostgreSQL tables store bulk enrichment reference data (DVF, UK Price Paid, OMI zones). Zone hierarchies for all 4 countries are imported from GADM shapefiles into the existing `zones` table.

---

## Technical Context

**Language/Version**: Python 3.12 (spiders + pipeline enrichers)  
**Primary Dependencies**: nats-py 2.6+, asyncpg 0.29+, httpx 0.27+, playwright 1.43+ (SeLoger, LeBonCoin), beautifulsoup4 4.12+ (Rightmove), parsel 1.9+ (existing), rapidfuzz 3.6+ (UK address matching), geopandas 0.14+ (zone import), shapely 2.x (existing), pydantic-settings 2.2+, structlog 24.x, pytest-httpx 0.30+ (spider tests)  
**Storage**: PostgreSQL 16 + PostGIS 3.4 (3 new tables + listings column extensions); Redis 7 (existing seen-IDs dedup)  
**Testing**: pytest + pytest-asyncio + pytest-httpx (unit); testcontainers (integration)  
**Target Platform**: Linux container (Kubernetes)  
**Performance Goals**: Each spider ≥1,000 listings/run; Funda ≤0.5 req/s; DVF spatial query <100ms  
**Constraints**: Funda 1 req/2s rate limit; DVF CSV bulk import ~2GB; UK Price Paid CSV ~5GB; OMI data semi-annual refresh  
**Scale/Scope**: 7 new spiders, 4 new enrichers, 4 country zone imports, 3 new DB tables, 7 new YAML mapping files

### 1.4 Zone Import Script Pattern

```python
# scripts/import_gadm_zones.py
import geopandas as gpd
import asyncpg

LEVEL_MAP = {
    "IT": {0: 0, 1: 1, 2: 2, 3: 3},   # country/regione/provincia/comune
    "FR": {0: 0, 1: 1, 2: 2, 3: 3},   # country/région/département/commune
    "GB": {0: 0, 1: 1, 2: 2, 3: 3},   # country/country/county/district
    "NL": {0: 0, 1: 1, 2: 2},          # country/provincie/gemeente (3 levels)
}

async def import_country(conn, country_code, gpkg_path):
    gdf = gpd.read_file(gpkg_path)
    gdf = gdf.to_crs(epsg=4326)
    
    for _, row in gdf.iterrows():
        level = LEVEL_MAP[country_code][row["GID_level"]]
        slug = slugify(f"{country_code}-{row['NAME_1']}-{row.get('NAME_2','')}")
        geom_wkt = row.geometry.wkt
        bbox_wkt = box(*row.geometry.bounds).wkt
        area_km2 = row.geometry.to_crs(epsg=3035).area / 1e6
        
        await conn.execute("""
            INSERT INTO zones (name, name_local, country_code, level, 
                               parent_id, geometry, bbox, area_km2, slug, osm_id)
            VALUES ($1,$2,$3,$4,$5,
                    ST_GeomFromText($6,4326),
                    ST_GeomFromText($7,4326),
                    $8,$9,$10)
            ON CONFLICT (slug) DO UPDATE
              SET geometry = EXCLUDED.geometry,
                  area_km2 = EXCLUDED.area_km2,
                  updated_at = NOW()
        """, row["NAME_1"], row.get("VARNAME_1"), country_code, level,
             parent_id, geom_wkt, bbox_wkt, area_km2, slug, None)
```
