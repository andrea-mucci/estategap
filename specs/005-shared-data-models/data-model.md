## Enumerations (shared across all models)

### `PropertyCategory` (Python: `str` enum | Go: `string` type)

| Value | Description |
|-------|-------------|
| `residential` | Apartments, houses, villas, studios |
| `commercial` | Offices, retail, mixed-use |
| `industrial` | Warehouses, factories, logistics |
| `land` | Undeveloped plots, buildable land |

DB column: `property_category VARCHAR(20)` in `listings`

---

### `DealTier` (Python: `IntEnum` | Go: `int` type)

| Value | Label | Description |
|-------|-------|-------------|
| `1` | Great Deal | Asking price significantly below estimated market value |
| `2` | Good Deal | Asking price moderately below estimated market value |
| `3` | Fair | Asking price close to estimated market value |
| `4` | Overpriced | Asking price above estimated market value |

DB column: `deal_tier SMALLINT` in `listings`

---

### `ListingStatus` (Python: `str` enum | Go: `string` type)

| Value | Description |
|-------|-------------|
| `active` | Listing is currently on the market |
| `delisted` | Listing removed from portal (reason unknown) |
| `sold` | Listing confirmed sold |

DB column: `status VARCHAR(20)` in `listings`

> **Note**: Existing stub has `withdrawn` and `expired`. These are replaced by `delisted` to align with the DB schema.

---

### `SubscriptionTier` (Python: `str` enum | Go: `string` type)

| Value | Alert Limit | Description |
|-------|-------------|-------------|
| `free` | 3 | No-cost tier with limited alerts |
| `basic` | 10 | Entry paid tier |
| `pro` | 50 | Mid-tier with full feature access |
| `global` | 200 | Multi-country access |
| `api` | unlimited | Programmatic API access |

DB column: `subscription_tier VARCHAR(20)` in `users`

## Python Models (`libs/common/estategap_common/models/`)

### Base: `EstateGapModel`

- `model_config = ConfigDict(strict=True, extra="forbid")`
- All `datetime` fields typed as `AwareDatetime` (Pydantic built-in — rejects naive datetimes)

### Shared Validators (module-level, importable from `_base.py`)

| Validator | Rule | Applied To |
|-----------|------|-----------|
| `validate_country_code` | Must be a valid ISO 3166-1 alpha-2 code (249-entry frozenset) | `country`, `country_code` fields |
| `validate_currency_code` | Must be a valid ISO 4217 alphabetic code (~170-entry frozenset) | `currency` fields |
| `validate_positive_price` | `Decimal > 0` | `asking_price` in `NormalizedListing` |
| `validate_positive_area` | `Decimal > 0` | `built_area_m2`, `usable_area_m2` in `NormalizedListing` |

## Go Structs (`libs/pkg/models/`)

### Module: `github.com/estategap/libs/models`

All structs use:
- `json:"<name>"` tags matching Python `model_dump(mode="json")` field names
- `db:"<name>"` tags matching DB column names
- `pgtype.UUID` for UUID fields (supports `Scan` from pgx rows)
- `pgtype.Timestamptz` for all timestamp fields
- `*decimal.Decimal` for all nullable price/numeric fields; `decimal.Decimal` for required ones

## Entity Relationships

```
Country (code PK)
  ├── Portal (country_code FK)
  └── Zone (country_code FK)
        └── Zone (parent_id self-FK, nullable)

Listing (id, country composite PK — partitioned by country)
  ├── Portal (portal_id FK, nullable)
  ├── Zone (zone_id FK, nullable)
  └── PriceHistory (listing_id FK)

User (id PK)
  ├── AlertRule (user_id FK)
  │     └── AlertLog (rule_id FK)
  └── ConversationState (user_id FK, nullable)
        └── linked to AlertRule (alert_rule_id FK, nullable)

ScoringResult — value object, not persisted independently
  (merged into Listing.estimated_price, deal_score, deal_tier, etc.)

Subscription — value object derived from User fields
  (no separate DB table; constructed from User record)
```

## State Transitions

### `ListingStatus`

```
[scraper emits] → active
active → delisted   (portal removes listing)
active → sold       (portal marks as sold / confirmed transaction)
delisted → active   (listing re-appears on portal)
```

### `ConversationState.status`

```
active → completed  (all criteria collected, alert saved)
active → abandoned  (session timeout or user leaves)
```
