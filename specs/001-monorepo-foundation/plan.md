# Implementation Plan: Monorepo Foundation

**Branch**: `001-monorepo-foundation` | **Date**: 2026-04-16 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-monorepo-foundation/spec.md`

## Summary

Scaffold the complete EstateGap monorepo from a blank repository: 10 microservice directories (6 Go, 4 Python), a shared Next.js frontend, Protobuf contracts with buf tooling, shared Go and Python libraries, a root Makefile, Helm chart scaffolding, and four GitHub Actions CI pipelines. Every service is empty-but-valid вЂ” it compiles, passes linting, and is covered by CI before any business logic is written.

## Technical Context

**Language/Version**: Go 1.23 (Go services + shared libs), Python 3.12 (Python services + shared libs), TypeScript 5.x / Node 22 (Frontend)
**Primary Dependencies**:
- Go: `google.golang.org/grpc`, `google.golang.org/protobuf`, `go.uber.org/zap` в†’ replaced by `log/slog`, `github.com/spf13/viper`, `github.com/nats-io/nats.go`
- Python: `pydantic>=2`, `nats-py`, `asyncpg`, `structlog`, `grpcio`, `grpcio-tools`
- Frontend: `next@15`, `react@19`, `tailwindcss@4`, `typescript@5`
- Tooling: `buf`, `golangci-lint`, `ruff`, `mypy`, `uv`

**Storage**: PostgreSQL 16 + PostGIS 3.4 (referenced in libs/common/db.py; no schema in this phase)
**Testing**: Go: `go test` (table-driven), Python: `pytest` + `pytest-asyncio`, Frontend: Vitest + React Testing Library
**Target Platform**: Linux (Docker + Kubernetes); developer workstations (macOS/Linux)
**Project Type**: Polyglot microservices monorepo
**Performance Goals**: Not applicable to scaffolding phase
**Constraints**: Go images < 20 MB; Python images < 200 MB; Frontend image < 100 MB; `make build-all` < 5 min on CI
**Scale/Scope**: 10 services + 1 frontend + 4 proto files + 4 CI pipelines + 1 Helm chart
