# API Contracts: Stripe Subscription Management

**Branch**: `008-stripe-subscriptions` | **Date**: 2026-04-17

Base path: all routes are under the `api-gateway` service. Authenticated routes require `Authorization: Bearer <access_token>`.

---

## POST /v1/subscriptions/checkout

**Auth**: Required (JWT Bearer)  
**Description**: Creates a Stripe Checkout Session for upgrading to a paid subscription tier. Returns a redirect URL to the Stripe-hosted payment page.

### Request

```json
{
  "tier": "pro",
  "billing_period": "monthly"
}
```

| Field | Type | Required | Values | Notes |
|-------|------|----------|--------|-------|
| `tier` | string | yes | `basic`, `pro`, `global`, `api` | Target subscription tier |
| `billing_period` | string | yes | `monthly`, `annual` | Billing cycle |

### Response `200 OK`

```json
{
  "checkout_url": "https://checkout.stripe.com/c/pay/cs_test_..."
}
```

### Error Responses

| Status | Code | Condition |
|--------|------|-----------|
| `400 Bad Request` | `invalid_tier` | Tier value not in allowed set |
| `400 Bad Request` | `invalid_billing_period` | Period value not in allowed set |
| `400 Bad Request` | `already_subscribed` | User already has an active/trialing subscription |
| `500 Internal Server Error` | `stripe_error` | Stripe API returned an error |

## POST /v1/webhooks/stripe

**Auth**: None (public endpoint; verified via Stripe signature)  
**Description**: Receives Stripe webhook events. Verifies the `Stripe-Signature` header using `STRIPE_WEBHOOK_SECRET`. Processes events idempotently.

### Request Headers

| Header | Required | Notes |
|--------|----------|-------|
| `Stripe-Signature` | yes | HMAC signature generated by Stripe |
| `Content-Type` | yes | `application/json` |

### Request Body

Raw JSON Stripe event payload (read as `[]byte` for signature verification — must not be parsed before verification).

### Handled Event Types

| Event Type | Action |
|-----------|--------|
| `checkout.session.completed` | Create `subscriptions` row; update `users.subscription_tier`, `stripe_customer_id`, `stripe_sub_id` |
| `customer.subscription.updated` | Update `subscriptions` row (tier, status, periods); update `users.subscription_tier` |
| `customer.subscription.deleted` | Set `subscriptions.status = 'cancelled'`; set `users.subscription_tier = 'free'` |
| `invoice.payment_failed` | Set `subscriptions.payment_failed_at`; enqueue grace-period downgrade in Redis |
| `invoice.payment_succeeded` | Clear `subscriptions.payment_failed_at`; remove pending downgrade from Redis |

### Response `200 OK`

```json
{}
```

### Error Responses

| Status | Code | Condition |
|--------|------|-----------|
| `400 Bad Request` | `invalid_signature` | Stripe-Signature header missing or HMAC mismatch |
| `400 Bad Request` | `invalid_payload` | Body cannot be parsed as a Stripe event |

> **Note**: Always return `200 OK` for unknown event types (Stripe expects 200 for events the endpoint chooses not to handle). Only return `400` for signature failures or unparseable payloads.

## GET /v1/subscriptions/me

**Auth**: Required (JWT Bearer)  
**Description**: Returns the current user's subscription status, tier, billing details, and next invoice date.

### Response `200 OK` — Free tier

```json
{
  "tier": "free",
  "status": "free",
  "billing_period": null,
  "current_period_end": null,
  "trial_end_at": null
}
```

### Response `200 OK` — Trialing

```json
{
  "tier": "pro",
  "status": "trialing",
  "billing_period": "monthly",
  "current_period_end": "2026-05-17T00:00:00Z",
  "trial_end_at": "2026-05-01T00:00:00Z"
}
```

### Response `200 OK` — Active

```json
{
  "tier": "pro",
  "status": "active",
  "billing_period": "annual",
  "current_period_end": "2027-04-17T00:00:00Z",
  "trial_end_at": null
}
```

### Response `200 OK` — Past Due (in grace period)

```json
{
  "tier": "pro",
  "status": "past_due",
  "billing_period": "monthly",
  "current_period_end": "2026-04-17T00:00:00Z",
  "trial_end_at": null
}
```

### Response fields

| Field | Type | Notes |
|-------|------|-------|
| `tier` | string | Current effective tier (`free`, `basic`, `pro`, `global`, `api`) |
| `status` | string | `free` / `trialing` / `active` / `past_due` / `cancelled` |
| `billing_period` | string\|null | `monthly` / `annual`; null for free tier |
| `current_period_end` | string\|null | ISO 8601 UTC; next invoice date; null for free tier |
| `trial_end_at` | string\|null | ISO 8601 UTC; null if not in trial |

## POST /v1/subscriptions/portal

**Auth**: Required (JWT Bearer)  
**Description**: Creates a Stripe Customer Portal session for self-service subscription management (upgrade, downgrade, cancel, update payment method).

### Request

No request body required.

### Response `200 OK`

```json
{
  "portal_url": "https://billing.stripe.com/p/session/live_..."
}
```

### Error Responses

| Status | Code | Condition |
|--------|------|-----------|
| `400 Bad Request` | `no_subscription` | User has no Stripe customer ID (never subscribed) |
| `500 Internal Server Error` | `stripe_error` | Stripe API returned an error |

## Route Registration Summary

```
POST /v1/webhooks/stripe          → StripeWebhook    (no auth)
POST /v1/subscriptions/checkout   → Checkout         (JWT auth required)
GET  /v1/subscriptions/me         → Me               (JWT auth required)
POST /v1/subscriptions/portal     → Portal           (JWT auth required)
```

> Routes `/v1/subscriptions/checkout` and `/v1/subscriptions/portal` are added to `mountAuthenticatedV1Routes` in `cmd/routes.go`. Route `/v1/webhooks/stripe` is already registered in `cmd/main.go` outside the auth group.
