# Implementation Plan: Kind Cluster & Helm Validation

**Branch**: `029-kind-helm-validation` | **Date**: 2026-04-17 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/029-kind-helm-validation/spec.md`

## Summary

Set up a local Kubernetes development environment using kind (3-node cluster) so every developer can run the full EstateGap platform in under 5 minutes. Deliver comprehensive Helm chart validation: `values.schema.json`, a 27-case helm-unittest suite, conformance checks, install/upgrade/rollback tests, and fixture-based seed data. Add `values-test.yaml` with test-mode doubles for all external dependencies.

---

## Technical Context

**Language/Version**: Bash (scripts), Python 3.12 (seed loader + conformance), YAML (kind config + helm tests), HCL (docker-bake.hcl), GNU Make  
**Primary Dependencies**: kind 0.24+, Helm 3.14+, helm-unittest plugin, kubectl 1.30+, docker buildx, pyyaml, asyncpg, boto3, redis-py  
**Storage**: PostgreSQL 16 + PostGIS 3.4 (seeded via asyncpg), MinIO (seeded via boto3), Redis 7 (seeded via redis-py)  
**Testing**: helm-unittest (YAML), pytest (Python conformance), Bash scripts (install/upgrade/rollback)  
**Target Platform**: Local developer machine (macOS/Linux), kind cluster on Docker  
**Project Type**: Developer tooling + infrastructure validation  
**Performance Goals**: `make kind-reset` completes within 5 min (excluding first-time build); seed data loads in < 30s  
**Constraints**: Must work offline (no external registry); no real Stripe/LLM/scraper calls in test mode  
**Scale/Scope**: 10 services + 1 frontend; 1,000 fixture listings; 27 helm-unittest test cases

## Implementation Phases

### Phase A: Kind Cluster Infrastructure

**Deliverables**:
1. `tests/kind/cluster.yaml` — 3-node kind config with all 8 port mappings (80, 443, 8080, 8081, 9090, 3001, 3000, 5432)
2. `tests/kind/registry-configmap.yaml` — containerd mirror for `localhost:5001`
3. `tests/kind/port-forward.sh` — background port-forward manager writing PIDs to `.kind-pids`
4. `tests/kind/cleanup.sh` — reads `.kind-pids`, kills all PIDs, then `kind delete cluster`

**Key decisions** (from research.md §2):
- Local registry on `localhost:5001` instead of `kind load` per worker (faster for 2+ workers)
- `kindest/node:v1.30.0` image pinned
- `ingress-ready=true` node label on control-plane for nginx ingress controller

---

### Phase B: Makefile Targets

**Deliverables**:
1. `mk/kind.mk` — all `kind-*` and `helm-*` Make targets per contracts/makefile-targets.md
2. `docker-bake.hcl` — BuildKit target per service, local cache (`type=local`)
3. Root `Makefile` updated: `include mk/kind.mk` line added
4. `.gitignore` updated: `.make-cache/`, `.kind-pids`, `.buildx-cache/`

**Key decisions** (from research.md §3, §4, §5):
- Hash-based change detection: `sha256sum Dockerfile + source dir` → `.make-cache/<svc>.digest`
- `docker buildx bake` for parallel builds
- `kind-reset` chains `kind-down kind-up kind-build kind-load kind-deploy kind-seed`

---

### Phase C: Helm Chart Additions

**Deliverables**:
1. `helm/estategap/values-test.yaml` — test profile per data-model.md §4
2. `helm/estategap/values.schema.json` — JSON Schema per data-model.md §5
3. `helm/estategap/templates/configmap.yaml` — `{{- if .Values.testMode.enabled }}` block injecting `ESTATEGAP_TEST_MODE=true`, `NOW_OVERRIDE`, `FAKE_LLM_PROVIDER=true`

**Key decisions** (from research.md §6, §7):
- Observability (prometheus/loki/tempo) disabled in test profile to save RAM
- Hand-written JSON Schema (not generated); `additionalProperties: false` at top level
- Schema uses `$defs` for reusable sub-schemas (service spec, observability component)

**Deliverables**: 8 YAML test files in `helm/estategap/tests/` with 27 total test cases

Per data-model.md §6:

### Phase E: Conformance & Install Tests

**Deliverables**:
1. `tests/helm/conformance.py` — parses `helm template` multi-doc YAML; asserts per FR-012
2. `tests/helm/install-test.sh` — `helm install` + `kubectl wait` per namespace + `/readyz` curl
3. `tests/helm/upgrade-test.sh` — package 0.1.0 + 0.2.0 locally, install/seed/upgrade/row-count-diff/rollback
4. `tests/helm/schema-test.sh` — `helm install --set global.imageRegistry=123` → expect schema error

**Key decisions** (from research.md §9, §13):
- `conformance.py` uses `yaml.safe_load_all()` for multi-document stream
- Upgrade test uses row counts (not pg_dump diff) to verify data integrity
- Both chart versions packaged locally to `dist/` directory

---

### Phase F: Seed Data Fixtures

**Deliverables**:
1. `tests/fixtures/load.py` — async seed loader using asyncpg (bulk copy), boto3 (MinIO), redis-py
2. `tests/fixtures/pyproject.toml` — standalone uv project
3. JSON fixture files: users.json (5), listings/ (200×5), zones/ (7 cities), alerts.json (10)
4. `tests/fixtures/conversations/sample_01.json`
5. `tests/fixtures/ml-models/generate.py` + 5 stub ONNX files (sklearn LinearRegression → skl2onnx)
6. `tests/fixtures/html-samples/` — 4 portals × 1-2 HTML pages each

**Key decisions** (from research.md §10):
- `asyncpg.copy_records_to_table` for bulk listing insert (fastest path)
- `INSERT ... ON CONFLICT DO NOTHING` on unique keys (idempotent loads)
- `load.py` connects via env vars: `PG_DSN`, `REDIS_URL`, `MINIO_ENDPOINT`, `MINIO_ACCESS_KEY`, `MINIO_SECRET_KEY`

### Phase G: Test Mode Integration (Service Stubs)

**Note**: This phase defines the interface; service-level implementations are separate tasks.

**ConfigMap contract** (from research.md §11):
```yaml
# When testMode.enabled=true, configmap.yaml injects:
ESTATEGAP_TEST_MODE: "true"
NOW_OVERRIDE: "{{ .Values.testMode.nowOverride }}"
FAKE_LLM_PROVIDER: "true"
TEST_SCHEDULE_OVERRIDE: "{{ .Values.testMode.scheduleOverride }}"
STRIPE_WEBHOOK_SECRET: "{{ .Values.testMode.stripeWebhookSecret }}"
FIXTURE_MINIO_BUCKET: "{{ .Values.testMode.fixtureMinIOBucket }}"
```

**Per-service stubs required**:
- `services/spider-workers/`: `FixtureSpider` — reads listing JSON from MinIO `fixtures/` bucket
- `services/api-gateway/`: accept `STRIPE_WEBHOOK_SECRET` from env for signature verification
- `services/ai-chat/`: `FakeLLMProvider` — returns deterministic canned responses
- `services/scrape-orchestrator/`: parse `TEST_SCHEDULE_OVERRIDE` as cron expression
- All services: check `NOW_OVERRIDE` env var; if set, use as current time

## Risk Register

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Port conflicts on developer machine | Medium | `kind-up` pre-checks ports; document required list |
| Docker resource limits too low | Medium | Pre-flight check via `docker info` for CPU/memory |
| helm-unittest plugin version incompatibility | Low | Pin plugin version in `make kind-prereqs` |
| ONNX stub model rejected by ml-scorer | Low | Validate with `onnxruntime.InferenceSession` in `generate.py` |
| Seed data schema drift from Alembic migrations | Medium | `load.py` validates column names at startup before bulk insert |
