# Implementation Plan: Listing & Zone Data Endpoints

**Branch**: `007-listing-zone-endpoints` | **Date**: 2026-04-17 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/007-listing-zone-endpoints/spec.md`

## Summary

Enhance and complete the `GET /v1/listings`, `GET /v1/zones`, and `GET /v1/zones/{id}/analytics` endpoints, and add five new endpoints (`GET /v1/listings/{id}` detail, `GET /v1/zones/compare`, `GET /v1/countries`, `GET /v1/portals`) — all within the existing Go API Gateway service. The work is entirely within `services/api-gateway/` plus minor additions to `libs/pkg/models/`. No new services are introduced.

## Technical Context

**Language/Version**: Go 1.23  
**Primary Dependencies**: chi v5.2.1, pgx v5.7.2, go-redis v9.7.0, shopspring/decimal v1.4.0  
**Storage**: PostgreSQL 16 + PostGIS 3.4 (read replica pool); Redis 7 (caching)  
**Testing**: Go table-driven tests with `testcontainers-go`  
**Target Platform**: Linux (Kubernetes pod)  
**Project Type**: HTTP web service (API Gateway)  
**Performance Goals**: < 500ms p95 for listing search with 100k rows and complex filters  
**Constraints**: All reads from replica pool; no direct service-to-service HTTP calls; structured `slog` logging; no ORMs — raw SQL via pgx  
**Scale/Scope**: 100k active listings per country; up to 50 active countries; up to 100 portals

## Phase 0: Research Findings

Full details in [research.md](research.md). Key decisions:

| Decision | Resolution |
|----------|-----------|
| Cursor strategy per sort | Composite base64 cursor `{float64_bits}|{uuid}` per sort field |
| Zone filtering | `l.zone_id = $zone_id` (pipeline-assigned); ST_Within as documented fallback |
| Currency conversion | SQL JOIN on `exchange_rates`; headers `X-Currency` + `X-Exchange-Rate-Date` |
| Subscription gating | SQL predicates from context tier; Basic country check via DB lookup |
| Redis caching | `cache.GetOrSet[T]` helper; zones/countries cached, listing search not cached |
| Zone analytics shape | Rewrite to `date_trunc('month', ...)` + `generate_series` for 12 fixed points |
| zone_statistics view | Assume exists; create migration if absent (prerequisite check at task start) |
| Response envelope | Change all list responses to `{data, pagination, meta}` (no backwards compat shim) |
| Route order | `/zones/compare` registered before `/zones/{id}` in chi |

## Implementation Notes

### 1. `cache/cache.go` — New File

```go
package cache

type Client struct{ rdb *redis.Client }

func (c *Client) GetOrSet[T any](ctx context.Context, key string, ttl time.Duration, fn func() (T, error)) (T, error) {
    // 1. GET from Redis → decode JSON → return if hit
    // 2. Miss: call fn(), encode JSON → SET with TTL → return
}
```

Pass `*cache.Client` to `ZonesRepo`, `ReferenceRepo` constructors. `ListingsRepo` does not use cache.

### 2. Repository: `SearchListings` Dynamic SQL

The existing dynamic SQL builder is extended. Key additions:

- **ZoneID filter**: `AND l.zone_id = $N`
- **PropertyType**: `AND property_type ILIKE $N`
- **Bedrooms**: `AND bedrooms >= $N`
- **Bathrooms**: `AND bathrooms >= $N`
- **PortalID**: `AND portal_id = $N`
- **DaysOnMarket**: `AND days_on_market BETWEEN $N AND $M`
- **FreeTierGate**: `AND first_seen_at < NOW() - INTERVAL '48 hours'`
- **AllowedCountries**: `AND country = ANY($N)`
- **Currency conversion**: LEFT JOIN exchange_rates, add `asking_price_eur * (1.0 / NULLIF(er.rate_to_eur, 0)) AS price_converted`

**Dynamic ORDER BY** (not parameterised — sort column is validated against an allowlist before string interpolation):
```go
sortColumn := map[string]string{
    "recency":        "first_seen_at",
    "deal_score":     "deal_score",
    "price":          "asking_price_eur",
    "price_m2":       "price_per_m2_eur",
    "days_on_market": "days_on_market",
}[filter.SortBy]  // validated; default "first_seen_at"
```

**Cursor for non-recency sorts**: Use `encodeFloatCursor(floatVal, id)` / `decodeFloatCursor(cursor)` returning `(float64, uuid.UUID, error)`. SQL condition: `(sort_col, id) < ($cur_val, $cur_id)` for DESC.

**Total count**: Run a separate `SELECT COUNT(*) FROM listings WHERE {same conditions without cursor and limit}`. Use `context.WithTimeout` with 200ms to fail fast if count is slow.

### 7. Handler: Subscription Gating

```go
func gatingFromContext(ctx context.Context, repo *repository.UsersRepo, userID uuid.UUID) (freeTier bool, allowedCountries []string, err error) {
    tier := ctxkey.String(ctx, ctxkey.UserTier)
    switch models.SubscriptionTier(tier) {
    case models.SubscriptionTierFree:
        return true, nil, nil
    case models.SubscriptionTierBasic:
        user, err := repo.GetUserByID(ctx, userID)
        if err != nil { return false, nil, err }
        return false, user.AllowedCountries, nil  // AllowedCountries field — check if it exists in model
    default:
        return false, nil, nil
    }
}
```

**Note**: `models.User.AllowedCountries` may not exist yet — check and add to `libs/pkg/models/user.go` if absent.

Key test cases:
- Free-tier gate: listings < 48h are excluded
- Basic-tier gate: listings outside allowed countries excluded
- Cursor pagination: page 1 + page 2 = no duplicates, no gaps
- `sort_by=deal_score`: NULL deal_score listings excluded
- Currency conversion: USD result ≠ EUR result; header present
- Zone analytics: always returns exactly 12 items; zero-fill months work
